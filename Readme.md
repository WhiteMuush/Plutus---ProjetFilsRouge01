
# Scripts de Synchronisation et d'Archivage PLUTUS

## Contexte du Projet

La société **PLUTUS** est une entreprise en pleine expansion dans le secteur des luminaires et de leur
vente en ligne. Elle souhaite mettre en place une solution permettant de synchroniser et d’archiver
automatiquement les fichiers de commandes générés sur un serveur Linux on-premise vers un serveur
Windows Server. L’objectif est d'assurer un transfert de fichiers sécurisé, un archivage efficace et une
gestion des sauvegardes de manière régulière. Les fichiers traités sur le serveur Linux devront être
déplacés vers un répertoire d'archivage avant d’être sauvegardés, garantissant ainsi une récupération
rapide en cas de besoin.
La société PLUTUS recherche une solution qui soit fiable, automatisée et facilement maintenable, tout
en respectant un niveau de sécurité élevé pour protéger les données des commandes et des archives.
La solution n’est pas déployée et devra l’être de manière automatisée.

## Objectifs

- Synchroniser automatiquement les fichiers de commandes du serveur Linux on-premise vers Windows Server
- Archiver les fichiers traités de manière sécurisée et efficace
- Gérer les sauvegardes régulières avec récupération rapide
- Assurer un déploiement entièrement automatisé
- Respecter les normes de sécurité élevées pour la protection des données


## Démarrage Rapide

1. Consulter le Document d'Architecture Technique (DAT)
2. Valider la configuration d'environnement
3. Exécuter les scripts de déploiement automatisé
4. Tester la synchronisation sur données de test
